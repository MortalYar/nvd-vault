# Политика безопасности

[English](SECURITY.md) · **Русский**

NVD Vault — это инструмент в области безопасности: он помогает пользователям проводить аудит инвентаря ПО на известные уязвимости. Мы серьёзно относимся к безопасности самого инструмента и приветствуем ответственное раскрытие любых проблем.

## Поддерживаемые версии

NVD Vault находится в активной ранней разработке. Исправления безопасности выпускаются только для последней версии в ветке `main`. Старые коммиты и pre-release теги не патчатся — перед отправкой репорта обновитесь до актуальной версии.

| Версия            | Поддерживается     |
| ----------------- | ------------------ |
| `main` (последняя) | :white_check_mark: |
| `0.1.x`           | :white_check_mark: |
| `< 0.1`           | :x:                |

После выхода стабильного релиза `1.0` политика будет обновлена и охватит более длительный период поддержки для последних минорных версий.

## Как сообщить об уязвимости

**Пожалуйста, не сообщайте об уязвимостях через публичные GitHub Issues, Pull Requests или Discussions.**

Используйте один из приватных каналов:

- **Предпочтительно:** [GitHub Security Advisories](https://github.com/MortalYar/nvd-vault/security/advisories/new) — приватный репорт, видимый только мейнтейнерам.
- **Альтернатива:** создайте минимальный публичный issue с заголовком *"Security contact request"* (без деталей), и мейнтейнер свяжется с вами приватно.

В репорте по возможности укажите:

- Описание уязвимости и потенциальные последствия
- Шаги для воспроизведения (минимальный proof-of-concept — идеально)
- Затронутую версию, хеш коммита или тег релиза
- Вашу ОС и версию Python
- Предлагаемые меры защиты, если они у вас есть

## Чего ожидать

| Этап                              | Ориентировочный срок                              |
| --------------------------------- | ------------------------------------------------- |
| Подтверждение получения           | В течение **72 часов** с момента репорта          |
| Триаж и оценка серьёзности        | В течение **7 дней**                              |
| Обновления статуса во время фикса | Не реже раза в **14 дней**                        |
| Релиз исправления (critical/high) | В течение **30 дней** после подтверждения, по возможности |
| Публичное раскрытие               | Согласовывается с автором репорта после выпуска фикса |

Если уязвимость **принята**, вас ждёт:

- Подтверждение и, при необходимости, запрос CVE через GitHub Security Advisories
- Патч в ветке `main` и выпуск нового тега
- Указание авторства в advisory и release notes (либо анонимно — по вашему желанию)

Если уязвимость **отклонена** (например, поведение признано вне зоны ответственности или это известное ограничение), вы получите письменное объяснение причины и сможете раскрыть информацию публично.

## Зона ответственности

**В зоне ответственности:**

- Код в этом репозитории (`nvd_vault/`, `app.py`, файлы сборки)
- Настройки по умолчанию и примеры из поставки
- Встроенный веб-интерфейс и IPC-слой между Python-бэкендом и встроенным браузером

**Вне зоны ответственности:**

- Уязвимости в сторонних сервисах, к которым обращается инструмент (NVD API, EPSS, CISA KEV) — сообщайте их соответствующим командам
- Уязвимости в сторонних зависимостях, которые невозможно эксплуатировать через NVD Vault — сообщайте мейнтейнерам соответствующих библиотек
- Проблемы, требующие у атакующего предварительного локального исполнения кода или прав записи в директорию vault-а пользователя
- Социальная инженерия, физические атаки, атаки на GitHub-аккаунт пользователя

## Safe Harbor

Мы не будем предпринимать юридических действий в отношении исследователей, которые:

- Добросовестно соблюдают эту политику
- Сообщают своевременно и не раскрывают информацию публично до согласованной даты
- Избегают нарушения приватности, уничтожения данных и деградации сервиса
- Тестируют только свои собственные инсталляции и данные

Спасибо, что помогаете делать NVD Vault и его пользователей безопаснее.
