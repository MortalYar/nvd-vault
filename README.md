# NVD Vault

Desktop-приложение для аудита уязвимостей по open-source-инвентарю. Принимает список продуктов в формате `inventory.json`, обращается к [NVD (National Vulnerability Database)](https://nvd.nist.gov/), генерирует **vault** в формате связанных Markdown-заметок и предоставляет встроенный просмотрщик с навигацией по wiki-ссылкам.

## Возможности

- **Сборка vault** из inventory.json с автоматическим определением vendor через NVD CPE Dictionary
- **Точный матчинг версий** через CPE 2.3 с учётом диапазонов (`versionStartIncluding`, `versionEndExcluding` и т. д.)
- **Связанная структура заметок:** для каждой CVE — отдельный документ, связанный wiki-ссылками с продуктами и типами слабостей (CWE)
- **Встроенный просмотрщик** с тёмной темой, sidebar-навигацией, фильтром, цветовой индикацией severity
- **Кликабельные wiki-ссылки** `[[CVE-...]]`, `[[product]]`, `[[CWE-...]]` — навигация между заметками без выхода из приложения
- **Информация из CISA KEV** (Known Exploited Vulnerabilities) — отметка активно эксплуатируемых уязвимостей
- **Экспорт vault в ZIP-архив** для архивации или передачи

## Технологический стек

- **Backend:** Python 3.13, без сторонних UI-библиотек
- **UI:** PyWebView (нативный WebView2 на Windows) + HTML/CSS/JS без фреймворков
- **Markdown:** [marked.js](https://marked.js.org/) для рендеринга
- **HTTP:** requests для NVD API 2.0

## Установка

```powershell
git clone https://github.com/MortalYar/nvd-vault.git
cd nvd-vault

# Создаём виртуальное окружение (Python 3.13)
py -3.13 -m venv venv
.\venv\Scripts\Activate.ps1

# Зависимости
pip install -r requirements.txt
```

## Запуск

```powershell
python app.py
```

Откроется окно приложения с тремя вкладками.

## Использование

Приложение состоит из четырёх вкладок: одиночное сканирование, сборка vault, просмотр и граф связей.

### Вкладка «Быстрое сканирование»

Точечная проверка одного продукта без сохранения результата. Введи имя продукта, версию и (опционально) vendor — приложение опросит NVD и покажет список CVE, затрагивающих указанную версию. Каждая уязвимость отмечена цветом по уровню severity, у активно эксплуатируемых видна метка KEV.

Эта вкладка удобна для быстрых ad-hoc проверок: «насколько критична версия X прямо сейчас».

### Вкладка «Сборка Vault»

Полноценный аудит инвентаря с сохранением результата на диск. Подготовь `inventory.json` (см. `examples/sample_inventory.json`):

```json
{
  "vault_name": "My ELK Stack",
  "products": [
    { "name": "kibana", "version": "8.19.9", "vendor": "elastic" },
    { "name": "logstash", "version": "8.19.5", "vendor": "elastic" }
  ]
}
```

Выбери файл inventory и папку, в которой создать vault, нажми «Собрать Vault». Приложение опросит NVD по каждому продукту и создаст структуру связанных Markdown-заметок:

```
my-vault/
├── meta.json
├── cves/
│   ├── CVE-2025-37731.md
│   └── ...
├── products/
│   ├── kibana.md
│   └── logstash.md
└── cwes/
    ├── CWE-287.md
    └── ...
```

Каждая CVE-заметка содержит YAML frontmatter с метаданными (severity, CVSS, KEV-статус, теги) и тело с описанием, ссылками на patches и wiki-ссылками `[[product]]` и `[[CWE-...]]` на связанные документы.

### Вкладка «Просмотр Vault»

Открой ранее собранный vault и листай его прямо в окне приложения. Поддерживаются:

- **Sidebar-навигация** по продуктам, CVE и CWE с цветовой индикацией severity
- **Фильтр по имени** — мгновенное сужение списка в боковой панели
- **Полнотекстовый поиск** по содержимому всех заметок с подсветкой совпадений (через SQLite FTS5)
- **Кликабельные wiki-ссылки** — переход между связанными заметками без выхода из приложения
- **Экспорт в ZIP** — упаковка vault в архив для передачи или архивации

### Вкладка «Граф связей»

Визуализация vault как интерактивного графа. Узлы:
- **Продукты** (синие плашки)
- **CVE** (круги, цвет по severity, красная обводка у активно эксплуатируемых)
- **CWE** (серые ромбы)

Рёбра показывают связи «CVE затрагивает продукт» и «CVE является экземпляром CWE». Поддерживаются:

- Перетаскивание узлов и панорамирование фона
- Zoom колесом мыши
- Hover по узлу — tooltip с метаданными
- Клик по узлу — переход к соответствующей заметке во вкладке «Просмотр Vault»
- Фильтры по уровню severity, отображение только KEV-уязвимостей, скрытие узлов CWE
- Кнопки центрирования и пересборки layout
